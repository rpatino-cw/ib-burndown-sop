1. Read the Burndown List

TypeFormatMeaning
Port Down C7.19 22/1 => S7.6.3 10/1 No signal between these two ports
Misconnect C2.17 13/2 should be → S2.4.2 9/1 but is → S2.4.2 9/2 Cable in wrong port
BER C7.19 22/1 => S7.6.3 10/1 Link up but degraded — dirty or damaged

Notation

C7.19 22/1 = switch C7.19, port 22, lane 1 (slot 1 on the optic)

=> = these two ports should be connected to each other

4. Port Orientation (Slot Flip)

Bottom-row optics are flipped. Top = 1|2. Bottom = 2|1. Wrong slot = misconnect.
QM9790 — Front View Top Row: ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐ │ 1 │ 2 │ │ 1 │ 2 │ │ 1 │ 2 │ │ 1 │ 2 │ └──────┘ └──────┘ └──────┘ └──────┘ Bottom Row: ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐ │ 2 │ 1 │ │ 2 │ 1 │ │ 2 │ 1 │ │ 2 │ 1 │ FLIPPED └──────┘ └──────┘ └──────┘ └──────┘ 22/1 = port 22, slot 1 22/2 = port 22, slot 2 Each slot has its own LED: GREEN/AMBER = up · OFF = down

5. Decision Trees

Eliminate variables in order. Don't clean blindly — figure out WHERE the problem is first.

Port Down

PORT DOWN → IR card: light coming out of fiber? │ ├─ No light → Problem is upstream (other end or cable) │ ├─ Check other side — light there? │ │ ├─ Yes → Bad cable. Swap it. │ │ └─ No → Optic or switch issue. Loopback test both sides. │ │ ├─ Loopback PASS → Both optics good. Cable is bad. Swap. │ │ └─ Loopback FAIL → Bad optic. RMA the failing side. │ └─ Light present → LED status after clean + reseat? ├─ LED up → Was dirty. Done. Mark the sheet. └─ LED still off → Optic sees light but won't link ├─ Loopback PASS → Cable or config issue. Try known-good cable. └─ Loopback FAIL → Bad optic. RMA.

BER (Bit Error Rate)

BER → Clean + reseat both sides │ ├─ BER clears → Done. Mark the sheet. │ └─ BER persists → Deep clean (IPA + canned air) both sides ├─ BER clears → Done. └─ BER persists → Swap cable ├─ BER clears → Bad cable. Replace permanently. └─ BER persists → Optic degraded. RMA.

Misconnect

MISCONNECT → Verify cutsheet says "should be" port is correct │ ├─ Cutsheet doesn't match → Don't touch it. Comment on sheet, flag to network. │ └─ Cutsheet confirms → Is the correct port already occupied? ├─ Empty → Move cable. Verify slot flip (top vs bottom row). Check LED. └─ Occupied → Chain misconnect. Trace that cable too. └─ May need to move 2-3 cables in sequence. Map out all moves before unplugging anything.
Slot flip reminder: If the burndown says port X/1 and you're on the bottom row, slot 1 is on the RIGHT, not the left.

6. Loopback Matrix

Side ASide BVerdict
PASS PASS Both optics good — bad cable or config
PASS FAIL Side B optic bad — RMA
FAIL PASS Side A optic bad — RMA
FAIL FAIL Both bad OR bad switch port — try new optics, if same result → escalate
Bad switch port: New optic + multiple loopbacks still fail on same port = switch port is dead. Escalate to networking.

7. Escalation

When

  • Loopback fails with new optic → bad switch port
  • Known-good cable still won't link after loopback PASS on both sides
  • Cutsheet doesn't match burndown entry

How

Comment on the Google Sheet row with: switch name, port, what you tested, LED result, IR card result. Tag network team.
